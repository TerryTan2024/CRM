# 邮件自动归集 MVP

这一版先跑通“邮箱进件 -> CRM 客户消息 -> 自动识别 -> 知识库依据 -> 待确认回复草稿”的闭环。

## 当前能力

- 复用 EspoCRM 原生 IMAP/SMTP 邮件收取能力，网易邮箱、腾讯企业邮箱、QQ 邮箱、阿里邮箱、Google Workspace、Microsoft 365 等都可以接入。
- 新增定时任务 `ProcessCustomerEmails`，扫描已归档邮件并生成客户消息。
- 自动写入客户类型、需求阶段、下一步动作、来源邮箱、来源邮件 ID。
- 从知识库文章中匹配报价、样品、规格、售后、交付等关键词，生成知识库依据。
- 生成回复建议，但状态固定为“待确认”，不会自动发送。

## 邮箱接入

1. 在邮箱服务商后台开启 IMAP/SMTP。
2. 使用授权码或应用专用密码，不要直接使用邮箱登录密码。
3. 在 CRM 后台配置群组邮箱或个人邮箱。
4. 确认 EspoCRM 的计划任务 Cron 已运行。
5. 启用或手动运行计划任务：`ProcessCustomerEmails`。

## 测试方法

服务器容器内可手动执行：

```bash
cd /var/www/html
bin/command run-job ProcessCustomerEmails
```

测试邮件建议包含这些关键词之一：

- 报价、价格、quote、price
- 样品、规格、sample、spec
- 售后、退货、投诉、return、complaint

执行后在“客户消息”中查看新记录，重点检查：

- 客户类型
- 需求阶段
- 下一步动作
- 知识库依据
- AI 回复建议
